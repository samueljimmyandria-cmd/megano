back_hand Reconnaissance palmaire

MediaPipe Hands détecte 21 points de repère sur la main en temps réel. Au lieu de comparer les coordonnées brutes (qui varient avec la distance et l'orientation), on extrait un vecteur de caractéristiques invariant :

  • Rotation : le repère est pivoté pour aligner l'axe poignet→index MCP sur X
  • Normalisation : toutes les distances sont divisées par la distance base pour ignorer l'échelle
  • Vecteur final : 5 longueurs de doigt + 1 largeur de paume + 4 angles interdigitaux

La similarité est une distance euclidienne normalisée, convertie en score 0–100 %.

face Reconnaissance faciale

face-api.js utilise un réseau de neurones convolutif pour produire un descripteur facial de 128 dimensions. Deux visages sont comparés par distance euclidienne : plus la distance est petite, plus ils se ressemblent.

Pour augmenter la robustesse en conditions difficiles, on applique :

  • Auto-contrast (percentiles 2 % / 98 %) sur l'image
  • Tentatives avec plusieurs niveaux de zoom (80 %, 60 %, 40 %, 30 %)
  • Deux détecteurs : TinyFaceDetector + SsdMobilenetv1

merge_type Fusion multimodale

Les deux scores sont additionnés (chacun max 1.0, total max 2.0). Un utilisateur est validé si :

  • Score total > 20 % ET
  • Écart avec le 2e candidat > 0.2 % (sinon ambigu)

Si l'un des seuils échoue, l'utilisateur est invité à confirmer avec un PIN de secours.

storage Stockage

Toutes les données biométriques sont stockées dans localStorage du navigateur :

// Structure utilisateur
{
  name: "Marie Dupont",
  pin: "1234",              // optionnel
  handVectors: [[...], ...], // 3 vecteurs géométriques
  faceDescriptors: [[...]],  // descripteur 128-d (optionnel)
  createdAt: 1234567890
}

Aucune image n'est sauvegardée, seulement les vecteurs mathématiques. Et rien ne quitte votre navigateur.

bug_report Le bug MediaPipe corrigé

L'erreur originale :

RuntimeError: Aborted(Module.arguments has been replaced
with plain arguments_)

Était causée par la recréation multiple de l'instance Hands à chaque changement de vue. Le module WASM de MediaPipe garde un état global ; le réinstancier le corrompt.

Fix appliqué : une seule instance hands globale créée au premier usage, et les Camera MediaPipe sont recréées à la place — pas l'instance Hands.

rocket_launch Limitations honnêtes

  • Démo éducative — pas pour production
  • La précision dépend de l'éclairage et de la caméra
  • Pas de protection anti-replay pour le visage (un attaquant avec une photo peut passer)
  • Les modèles (~10 Mo) se téléchargent depuis un CDN à la première visite
